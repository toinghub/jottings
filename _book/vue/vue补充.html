vue 补充

vue 中动态引入图片为什么要是 require

为什么无法编译？

动态引入时,编译的是 src 后面的变量，就导致与图片资源编译过后的资源地址不一致， 导致无法正确的引入资源

为什么加上 require 就好了?

通过 require 方法拿到的文件地址，是资源文件编译过后的文件地址

静态的引入时，为什么返回的是编译过后的文件地址？

在 webpack 编译的 vue 文件的时候，遇见 src 等属性会默认的使用 require 引入资源路径
